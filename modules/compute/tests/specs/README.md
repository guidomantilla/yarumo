# Test Specifications — modules/compute/

## Flow

```
modules/compute/prompts/CORRECTNESS.md        (prompt)
    → modules/compute/CORRECTNESS.md           (mathematical correctness analysis)
        ↓ (input)
modules/compute/prompts/ACCEPTANCE_TESTS.md   (prompt)
    → ACCEPTANCE_TESTS.md                        (pure spec, no code, language-agnostic)
        ↓ (extracted once, maintained manually)
acceptance/00-07.md                              (Go codegen prompts)
    → modules/compute/tests/acceptance/*_test.go
```

## Files

### ACCEPTANCE_TESTS.md

Generated by the prompt `modules/compute/prompts/ACCEPTANCE_TESTS.md`.

Pure specification — text, tables, mathematical derivations, no code.
Reviewable by a domain expert independent of implementation language.

If this file changes, the codegen prompts in `acceptance/` must be updated manually.

### acceptance/

Go code generation prompts. Each file generates one or more `_test.go` files
in `modules/compute/tests/acceptance/`.

| Prompt | Generates | Description |
|--------|-----------|-------------|
| 00-CONTEXT.md | (shared context) | Project structure, imports, coding standards |
| 01-HELPERS.md | helpers_test.go | Tolerances, builders, assertion helpers |
| 02-INVARIANTS-LOGIC.md | invariants_logic_test.go | Sections 1.1-1.5: logic, predicate, temporal |
| 03-INVARIANTS-NUMERIC.md | invariants_numeric_test.go | Sections 1.6-1.7: fuzzy axioms, stats |
| 04-INVARIANTS-ENGINE.md | invariants_engine_test.go | Sections 1.8-1.12: deductive, bayesian, fuzzy, causal, mcdm |
| 05-GOLDEN.md | golden_scenario_test.go + golden_files_test.go | Sections 2-3: cross-paradigm scenario, behavioral snapshots |
| 06-ERRORS.md | error_contracts_test.go | Section 4: error contracts |
| 07-PERFORMANCE.md | performance_test.go | Section 5: performance baselines |

### Execution order

01-HELPERS.md must be executed first (others depend on helper definitions).
02-07 are independent of each other and can be executed in parallel.
